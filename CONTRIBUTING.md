# Contributing to TGArchive

Thank you for your interest in contributing. Please read the guidelines below before
opening an issue or a pull request.

## Reporting issues

If you encounter a bug or have a suggestion, please open an
[issue](https://github.com/Dxx-OTG/TGArchive/issues) and include:

- A clear description of the problem or proposal.
- Steps to reproduce the issue (for bugs).
- Your operating system and Python version.
- Any relevant log output.

**Do not include secrets** such as bot tokens, API credentials, or `.session` files in
issues, logs, or screenshots.

## Proposing changes

1. Fork the repository and create a branch from `main`
   (e.g. `fix/floodwait-handling` or `feature/export-format`).
2. Keep each pull request focused on a single change; smaller pull requests are easier
   to review and merge.
3. Make sure the project still runs correctly and follows the existing code style.
4. Open a pull request describing what the change does and why, referencing any related
   issue (e.g. `Closes #12`).

## Discussing first

For larger changes or new features, please open an issue to discuss the approach before
investing significant work. This avoids duplicated or misaligned effort.

## Security

Please do not report security vulnerabilities through public issues. See
[SECURITY.md](SECURITY.md) for responsible disclosure instructions.
